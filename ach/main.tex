\section{まえがき}
この文書はNo Brakes Gamesが制作、Curve Gamesが販売を行うゲームHuman Fall Flatの実績についての情報を集めたものである。この文書は常に正しい情報を保証するものではない。また、情報についてはSteam版をもとに作成している。
\section{実績の基礎知識}
実績とは、プレイヤーが起こしたアクションを認めるシステムである。実績の内容(条件)はゲームの制作者が設定し、実績の条件を満たすことを「解除」と呼ぶ。
Human Fall Flatでは、実績を管理するためにC\#クラスを用意している。
\subsection{分類}
\begin{description}
    \item[General] 条件を満たすことでいつでも解除することができる、一般的な実績。特定のステージでないと解除できないものが多い。
    \item[Stat-Based] 追跡システムを用いてプレイヤーの行動を記録し、一定条件に達することで解除できる実績。どんな場所でも解除できる。
\end{description}

\subsection{実績の解除システム}
ここでは、ゲーム内で実績を解除する時にどんな処理を行っているかについて書く。ここでは、「段差にご注意！」を例に解説する。
\begin{enumerate}
    \item ステージのScene内に実績のためのオブジェクトが生成される。(ここでは\mono{Intro/Level/Achievements/JumpGapAchievement}、また子オブジェクトに\mono{EntryCollider}、\mono{ZoneCollider}、\mono{PassCollider}。親オブジェクトはクラス\mono{JumpGapAchievement}のインスタンスでもある。)
    \item メソッド\mono{StatsAndAchievements.UnlockAchievement(Achievement.ACH\_INTRO\_JUMP\_GAP,\ false,\ -1)}が呼び出される。
\end{enumerate}
\section{実績の解説}
Stat-Basedは実績リストの途中に挿入されているが、今後の更新で増えることはおそらくなく、特殊な分類なので先に取り上げる。また、番号は列挙型\mono{Achievement}の順番に基づく。
\subsection{Stat-Based}
Stat-Based系の実績は\mono{StatsAndAchievements.UnlockAchievement}を経由しないことが判明した。よって、ここでは解除の条件と各(その実績に関係する)Statの増加方法についてのみ述べる。
\subsubsection{12.旅行者、13.旅人、14.冒険家}
\begin{itemize}
    \item 条件:\ \mono{int\ StatsAndAchievements.savedTravelM}が目標値に達する。
    \item 旅行者では1000、旅人では15000、冒険家では25000。
    \item \mono{Stat(savedTravelM)}の増加方法(原理)
    \begin{enumerate}
        \item プレイヤーの\mono{State}が別の状態から\mono{Walk}になったフレームで\mono{DontDestroyOnLoad/Player(Clone)/Ball}の位置を開始位置\mono{Vector3\ StatMonitorHuman.walkStartPos}とする。
        \item \mono{float\ UnityEngine.Time.deltaTime}秒毎に(毎フレーム)\mono{walkStartPos}から現在位置\mono{Vector3\ Human.instance.GameObject.transform.position}の差のY成分を0にし、大きさ\mono{float\ magnitude}を求める。
        \item プレイヤーの\mono{State}が\mono{Walk}から別の状態になったフレームで、\mono{magnitude}が0.5m以上なら\mono{float\ travelM}に加算する。0.5m未満なら何もしない。
        \item \mono{Walk}中に\mono{magnitude}が10を超えた場合、即座に\mono{magnitude}を\mono{travelM}に加算し、\mono{walkStartPos}を現在位置にする。
        \item \mono{travelM}の百の位が1以上増加した時点で、\mono{trunc(travelM)}を\mono{int\ savedTravelM}に保存する。
    \end{enumerate}
    \item 関連するクラス
    \begin{itemize}
        \item \mono{Human}
        \item \mono{StatMonitorHuman}
    \end{itemize}
\end{itemize}
\subsubsection{15.まるで雨のようだ}
\begin{itemize}
    \item 条件:\ \mono{int\ StatsAndAchievements.fall}が1000に達する。
    \item \mono{Stat(fall)}の増加方法(原理)
    \begin{enumerate}
        \item クラス\mono{HumanAPI.FallTrigger}をコンポーネントに持ち、\mono{FallTrigger.fallAchievement}が\mono{True}に設定されている有効な死亡判定に\mono{Player(Clone)}の特定部位が入ると、\mono{Game.Fall}が呼び出され、\mono{fall}が1増える。同じフレームで2個以上のパーツが死亡判定に入ったときは、すべてのパーツに対して1増加させる。
        \item \mono{fall}の値が増えるたびに値を保存する。
    \end{enumerate}
    \item 死亡判定の\mono{GameObject}におけるLayerの値によって挙動が変化する。
    \begin{itemize}
        \item \mono{Triggers}の場合は\mono{DontDestroyOnLoad/Player(Clone)/Ball}、\mono{DontDestroyOnLoad/Player(Clone)/Ball/Quick\ Volume\ (override\ focus)}、\mono{Player(Clone)/Ball/Ragdoll(Clone)/Hips/Waist/Chest/Head/Listener}の3つにのみリスポーンを発生させる。
        \item \mono{Default}など他のLayerに設定されている場合、親オブジェクトにクラス\mono{Human}を持つコライダー付きの子(\mono{Ball}から見て)オブジェクト全てが反応する。そのため、RTAではこれを利用し、一度に多くの\mono{fall}カウントを増やしている。
    \end{itemize}
    \item \mono{Layer}が\mono{Triggers}以外に設定されている死亡判定(最終更新:Viking)
